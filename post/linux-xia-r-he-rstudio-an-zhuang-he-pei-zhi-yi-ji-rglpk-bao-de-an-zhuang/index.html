新问题

Rglpk包无法安装

安装即出现报错

怀疑可能是权限的问题?

看网上也有不少人在问，见这个帖子

GLPK: No such file or directory error when trying to install R package

其实

sudo apt-get install libglpk-dev

这个方法很有可能是对的，只是因为我没有这个apr工具
如果要安装apt这个工具，请看centos安装apt-get

c

Zhiying Cui的答案是对的，但是很繁琐，而且在并行的代码中，由于foreach的.package参数需要传入包，这个时候就会出问题

所以请在终端输入命令

yum install glpk-devel

在Rstudio中输入

install.packages("Rglpk")

一切就ok了

一些tip

1另外，linux对大小写敏感，所以当我们source("xxx.R")的时候，注意下后缀名.R的大小写和实际文件名是否相同，不同的话，就会报错的

2关于权限和所属的问题的问题

如果出现无法读取文件，或者修改R code 无法保存，必须保存为副本的情况，则按如下操作

由于我的Rstudio的用户是xu，所以

chmod 777 /home/xu/DEAF  #赋予读写权限

usermod -g root xu # 赋予xu用户权限

sudo chown -R xu:xu DEAF #把所属改为xu用户

chown root /usr/lib/systemd/system/rstudio-server.service # 这个软件所属改为root

最好重启下，方便生效
